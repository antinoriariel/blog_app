# Calidad y criterios de aceptación

## Calidad funcional
- Cada Markdown válido debe producir una entrada HTML predecible.
- Los listados deben mostrar título, resumen, fecha y tags cuando existan.
- Las entradas borrador no deben aparecer públicamente.
- El orden de publicación debe ser estable y comprensible.

## Calidad visual
- La portada debe tener un punto focal claro.
- El contenido debe leerse con comodidad en pantallas pequeñas.
- Las tarjetas y secciones deben mantener coherencia visual.
- Las animaciones deben sumar sin distraer.

## Calidad técnica
- La validación debe detectar metadatos faltantes.
- El build local debe poder repetirse desde cero.
- Los assets deben quedar organizados y fáciles de servir.
- La estructura del código debe permitir crecer sin romper el contenido.

## Criterios de aceptación inicial
- Existe documentación completa del proyecto.
- El formato de entrada está definido.
- La arquitectura del generador está documentada.
- El sistema visual está especificado.
- El flujo de despliegue está claro.
